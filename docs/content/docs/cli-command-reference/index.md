The Galasa CLI (galasactl) is a command-line tool that helps you manage and run Galasa tests. This section guides you through installing the CLI, setting up your local environment, and running your first tests.

## What you'll learn

- Installing the required software prerequisites
- Downloading and installing the Galasa CLI
- Initializing your local Galasa environment
- Creating and running Galasa test projects
- Viewing test results

## Next steps

1. Review the [Prerequisites](./cli-prereqs.md) to ensure you have the required software installed
2. Follow the [Installing the Galasa CLI](./installing-cli-tool.md) guide to set up the CLI tool
3. [Initialize your local environment](./initialising-home-folder.md) to prepare for running tests
4. Create a Galasa project by following [Creating a Galasa project using the command line](./setting-up-galasa-project.md)
5. Run your first test by following [Running a Galasa test using the command line](./runs-submit-local.md)

## Working without network access

If your environment has restricted access to Maven Central or Docker Hub, consider using [Galasa offline](../using-galasa-offline/index.md). The offline distribution provides a self-contained package that can be shared across your organization without requiring external network connectivity.
